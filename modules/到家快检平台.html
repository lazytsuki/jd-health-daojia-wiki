到家快检及上门服务平台核心模块系统剖析
V4.0
主线脉络
互联网医院检测 + 护士/护工上门服务 + 平台模式全国化转型。V4.0核心变化：迪拜会议后从自营模式调整为平台模式，引入服务站站长角色，提前实现服务资源计费结算能力，新增数据看板和人工调度模块。

模块一 · C端用户域
核心目标
让用户高效完成上门检测/护理服务的下单、预约、改约、取消与退款。
业务身份与下单模式
业务身份入口履约模式关键特征
小鸟快验小程序（检测）派单制，提前1天预约护士接单生成服务码，样本绑定条码
金牌护士APP（护理）抢单制，即时/预约多次核销（1 SKU服务N次），医患签字
自营/非自营护士平台统一入口派单制排期数据来源不同
两类服务核心差异
对比项上门检测上门护理
商品金额含服务费+护士上门费+耗材，一次结清耗材包单独付费（建议整合）
采样方式护士采样，绑定样本条码寄送实验室护士采样，医患签字确认
预约方式小鸟快验仅预约次日，按小时排期无严格围栏，按三级城市范围
履约完成标志样本寄出+实验室确认医患签字+服务记录上传
一品多次否是（1 SKU上门N次，逐次核销）
关键业务准则
  • ・用户支付成功即生成服务单，无需用户操作；一个服务单可能对应多个样本条码
  • ・退款：护士接单前全额；护士上门后不支持主动取消，走投诉/仲裁；样本已寄出不支持退款
  • ・围栏校验：呼吸道检测有地理围栏限制，围栏内无护士资源则阻止下单
  • ・实验室匹配：下单时按检测项目实时匹配可服务的实验室
模块二 · 护士/服务者域
核心目标
建立可管控的护士入驻→服务→结算全链路，确保平台模式下的服务质量与成本可控。
入驻流程（三审四核）
阶段审批方审批内容
身份审核质控（ERP竞签）执业信息、身份证、资质证照片
技能认证运营（ERP竞签）服务项目点亮（技能=服务项目）
开放城市校验系统注册时校验城市是否开通
关键约束：一个Pin多个角色（护士+护工+康复师）共享同一Pin；人员标签（全职/兼职）打在Pin维度；全职运营统一排期，兼职自主排期。

耗材管理体系
核销逻辑：派单前预占 → 接单后核销(-n) → 取消时释放(+n)
警示规则：任意耗材≤3时，护士端顶部提示"耗材不足，请及时采购"
结算规则
结算节点：服务完成 → 生成"冻结"结算单
解冻条件：护士上传"医疗废物垃圾"处理照片
账期：解冻后3天可提现，平台扣手续费
一品多次：按次结算；耗材单独付费：分两笔结算单（服务费+耗材费）
模块三 · 运营管理域
核心目标
搭建可配置的运营基础设施，实现服务站·护士·商品·实验室·耗材的全链路管控。
配置层次全景
平台层：开放城市管理 / 标品库（国家标准） / 实验室管理 / 指标管理
服务站层：服务站管理（围栏/营业时间/达达门店）→ 服务站商品管理 → 服务站服务资源管理（全职护士绑定到站）
商品层：商品项目管理（SKU+业务身份+检测项目）→ 加项管理（主SKU↔子SKU）→ 活动管理（爆单调控/时段费减免）
护士层：服务资源管理（入驻审核/技能认证/标签）→ 排期日历 → 耗材管理
服务单拆分规则（核心业务准则）
  • ・一单一品：多SKU订单按一品一单拆解
  • ・检测项目最小单元：1检测项目 = 1服务单 = 1耗材包 = 1护士 = 1实验室
  • ・多实验室支持：不同检测项目可送检不同实验室
  • ・报告聚合：按检测人维度汇总所有检测结果
实验室匹配策略（可配置优先级）
优先级选项：整单（覆盖项目最多）→ 时效（距离最近） → 成本（结算价最低） → 兜底（创建时间最早）
Promise出报告时间：上门骑行时间 + 服务时长 + 回寄时间 + 检测时长
关键配置生效规则（系统稳定性关键）
商品/实验室项目修改时：已支付订单按快照履约，新支付订单按新数据。所有快照必须存储在订单/服务单/检测单上。

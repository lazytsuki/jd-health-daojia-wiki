服务者结算域

三层服务架构 · 算价引擎 · 随单结算 · 保底工资
结算域护士端运营端算价服务一期进行中
  • 核心升级：算价服务不再依赖C端结果，直接读取护士结算配置规则，实现BC解耦
  • 适用范围：所有护士检测/护理的履约单，包括200号单、4号单、外部无订单等
  • 触发时机：用户/外部接口下单，派单前即完成算价
二、核心算价公式
订单算价 = (服务费 + 费项 + 派单加价) - 平台服务费 - 渠道佣金 + 耗材费 + 平台补贴
费项 = 时段费 + 上门费 + 动态调整费
平台服务费 = (服务费 + 费项 + 耗材费 + 派单加价) × 系数
平台补贴 = 平台服务费 × 系数
三、护士类型对比
自营（全职）护士
  • 薪资结构：保底 + 绩效 + 补助
  • 结算周期：月结（次月10日）
  • 账期：T+10
  • 时间受平台管控
  • 派单履约率影响保底资格
兼职护士
  • 薪资结构：按单结算
  • 结算周期：日结
  • 账期：无
  • 自主决定接单时间
  • 对标美团众包模式
四、渠道合作四种结算方式
  • 方式1：仅结算给渠道，护士端无需展示金额
  • 方式2：护士结算一部分 + 渠道收佣金，护士端需展示扣减金额
  • 方式3：全部结算给护士，与自主招募一致
  • 方式4：渠道订单按方式1/2/3，其余同自主入驻
渠道标签在护士入驻时即需打上，识别标签后根据配置规则给护士或渠道结算。耗材也需按渠道×项目单独配置。
五、随单结算时点规则
履约节点SKU服务费费项（时段/上门/动态）派单加价平台服务费平台补贴

护士开始服务50%100%100%不结算开始结算

六、保底工资计算（自营护士）
次月1号计算：
if (派单履约率 ≥ 90% or 上月接单数 ≥ 150):
  → 护士上月工资 = max(订单总金额, 保底工资) 【需插入调账差额】
else:
  → 护士上月工资 = 订单总金额

保底调账 = 保底工资 - 护士上月订单总金额
派单履约率 = 履约完成订单 / 派单总数
  • 保底按城市等级配置（本期需在ducc配置）
  • 预留护士等级×城市等级叉乘配置逻辑
  • 绩效按单结算，补助暂时走手工调账
